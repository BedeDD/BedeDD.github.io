Placeholder für inputs

5 September 24

I was working on a feature for a customer's website and added new fields to the existing form. I wanted to add meaningful placeholders, so I used the option placeholder: t('activerecord.palceholders.ward.line1').
That primarily served the purpose, but it also made me think That's quite a lot of writing for something so relatively simple! The second thought was That would be a feature for Rails. So I first looked at the code in the FormHelper and what options it offers.
Rails wouldn't be Rails if it hadn't thought of such a fundamental feature.

ActionView::Helpers::Tags::Placeholderable is the name of the thing.
Instead of a String using true as value for the placeholder option, the attribute name is automatically used as a placeholder.

  • <%= form.text_field :foo, placeholder: true %>
  • Lookup: helpers.placeholder.{MODE_NAME}.{ATTRIBUTE_NAME} || activerecord.attributes.{MODEL_NAME}.{ATTRIBUTE_NAME}

For this example the placeholder for <%= form.text_field :line1, placeholder: true %> in the model Ward is expected under de.helpers.placeholder.ward.line1 in your locales.
